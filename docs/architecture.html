1. Tổng quan Kiến trúc

Hệ thống được thiết kế theo kiến trúc microservices với 6 thành phần chính, vận hành trên nền tảng container hóa Docker. Mỗi thành phần có trách nhiệm riêng biệt, giao tiếp qua REST API và database SQLite centralized.

DDM Engine là module chuyên biệt cho iOS 27+ Declarative Device Management. Nó sinh các JSON declarations với conditional activations.

7. Deployment Architecture

7.1 Docker Compose (Production)

services: backend → Node.js 22, Express, SQLite frontend → Nginx reverse proxy + static files backup → SQLite backup định kỳ (hàng ngày) monitor → Glances system monitoring Networks: iosperf-net (bridge) Volumes: backend-data (persistent), backend-logs Ports: 3100 (API), 80/443 (Web), 61208 (Monitor)

7.2 CI/CD Pipeline (GitHub Actions)

Jobs: 1. lint → ESLint 2. test → Node --test 3. profile-val → Validate mobileconfig XML 4. build → Docker build + push to GHCR 5. deploy → SSH deploy to production (on release) 6. pages → Deploy to GitHub Pages

7.3 Production Requirements

  • CPU: 2 cores (4 recommended)
  • RAM: 2 GB (4 GB recommended)
  • Storage: 20 GB SSD
  • OS: Ubuntu 22.04+ / Debian 12+
  • Dependencies: Docker 24+, Docker Compose 2.24+, Nginx
  • SSL: Let's Encrypt (certbot) hoặc SSL từ CDN

8.2 Health Checks

  • Backend: GET /api/v1/health — returns DB stats, uptime
  • Docker: HEALTHCHECK every 30s — wget to health endpoint
  • Database: WAL checkpoint auto, daily backup via cron

8.3 Backup Strategy

  • Daily: SQLite database backup (docker backup service)
  • Retention: 30 days
  • Restore: Copy backup file to data volume, restart container

8.4 Alerting (Recommended)

  • Uptime: Better Uptime / UptimeRobot — check health endpoint mỗi 5 phút
  • Error rate: Winston error log → Sentry / Datadog
  • Disk usage: 80% threshold → email alert
